Every device that connects to the Seccion9 VPN needs a `.conf` file generated by the server. Once you have that file, connecting takes a single command or a few clicks in a GUI.

## How clients work

When a new client is registered on the server, WireGuard generates a unique key pair for that device and assigns it a private IP address inside the `10.0.0.0/24` VPN subnet. The resulting `.conf` file bundles everything the client needs to establish the tunnel:

```ini client.conf.example
[Interface]
PrivateKey = <CLAVE_PRIVADA_CLIENTE>
Address = 10.0.0.X/24
DNS = 8.8.8.8

[Peer]
PublicKey = <CLAVE_PUBLICA_SERVIDOR>
Endpoint = <IP_PUBLICA_VPS>:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

| Field | Description |
|---|---|
| `PrivateKey` | The client's private key — unique per device, never shared |
| `Address` | The VPN IP assigned to this client (e.g. `10.0.0.2/24`) |
| `DNS` | DNS server used while the tunnel is active |
| `PublicKey` (Peer) | The server's WireGuard public key |
| `Endpoint` | The server's public IP and WireGuard port (`51820/UDP`) |
| `AllowedIPs` | `0.0.0.0/0` routes all traffic through the VPN |
| `PersistentKeepalive` | Sends a keepalive every 25 seconds to maintain NAT mappings |

## Getting a configuration file

There are three ways to generate a `.conf` file for a new client:

<Tabs>
  <Tab title="Web panel (recommended)">
    Open the management panel at `http://YOUR_VPS_IP:8443`, click **Add client**, enter a name, and download the `.conf` file or scan the QR code. The panel handles key generation and IP assignment automatically.
  </Tab>
  <Tab title="add-client.sh">
    Use this script when you already have the client's public key (generated on the client device):

    ```bash
    sudo bash onboarding/add-client.sh <name> <public_key> <assigned_ip>
    ```

    Example:

    ```bash
    sudo bash onboarding/add-client.sh alice CMvCw4gSxdv5xwkgx...= 10.0.0.2
    ```

    The script registers the peer live on the server and writes the `[Peer]` block to `wg0.conf` without a full WireGuard restart.
  </Tab>
  <Tab title="wg-manager.sh">
    The interactive terminal manager offers the same functions as the panel over SSH:

    ```bash
    sudo ./wg-manager.sh
    ```

    Select option **1 — Add client** from the menu. The script prompts for a name, generates keys, assigns an IP, and saves the configuration.
  </Tab>
</Tabs>

## Supported platforms

<CardGroup cols={2}>
  <Card title="Linux" icon="terminal" href="/clients/linux">
    Automated setup script or manual install using `wg-quick`. Supports systemd for auto-start on boot.
  </Card>
  <Card title="Windows" icon="windows" href="/clients/windows">
    PowerShell setup script or GUI import. Installs WireGuard automatically if not present.
  </Card>
  <Card title="macOS" icon="apple">
    Download the WireGuard app from the Mac App Store, then import the `.conf` file via **File → Import Tunnel(s) from File**.
  </Card>
  <Card title="iOS / Android" icon="mobile">
    Open the WireGuard app, tap **+**, and scan the QR code from the management panel. No manual file transfer needed.
  </Card>
</CardGroup>

## Choosing a protocol

Seccion9 supports two VPN protocols. Choose based on the network environment the client is connecting from:

| Client environment | Recommended protocol |
|---|---|
| Home network, cloud, pfSense / OPNsense / Mikrotik firewall | WireGuard (UDP 51820) |
| Restrictive corporate firewall (Cisco ASA, Fortinet, SonicWall) | OpenVPN (TCP 443) |

<Note>
  WireGuard over UDP 51820 is the default and preferred option. It is faster, simpler, and produces smaller configuration files. Only fall back to OpenVPN TCP 443 if UDP traffic is blocked at the network perimeter.
</Note>
